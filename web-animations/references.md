# Web Animation References

Tools, libraries, and resources for creating animations destined for the web. Update as new resources are found.

---

## Design & Motion Tools (visual / GUI)

- **Cavalry** — 2D motion design app built for procedural and data-driven animation. Strong for generative/parametric work, exports to web-friendly formats.
  https://cavalry.scenegroup.co

- **Rive** — Interactive animation tool with a lightweight web runtime. Design stateful animations (hover, scroll, click) and export directly to a <canvas> or WebGL embed.
  https://rive.app

- **Spline** — Browser-based 3D design tool. Build and animate 3D scenes, export as interactive web embeds or lightweight exports.
  https://spline.design

- **LottieFiles** — Platform for Lottie animations (JSON-based, originally from After Effects via Bodymovin). Huge library of premade animations, editor, and lightweight web player.
  https://lottiefiles.com

- **SVGator** — GUI tool for creating SVG animations without code. Exports clean CSS/JS-animated SVGs.
  https://www.svgator.com

## JavaScript Animation Libraries

- **GSAP (GreenSock)** — Industry-standard JS animation library. ScrollTrigger plugin is excellent for scroll-driven animations. Extremely performant, wide browser support.
  https://gsap.com

- **Framer Motion** — Declarative animation library for React. Great DX for layout animations, gestures, and scroll-linked effects.
  https://motion.dev

- **Anime.js** — Lightweight JS animation library. Clean API for CSS properties, SVG, DOM attributes, and JS objects.
  https://animejs.com

- **Three.js** — 3D library for WebGL. For when animations need to go full 3D — particle systems, shaders, immersive scenes.
  https://threejs.org

## CSS-Native Animation

- **CSS scroll-timeline / animation-timeline** — Native CSS spec for scroll-driven animations. No JS required. Browser support is growing (Chrome/Edge stable, Firefox behind flag).
  https://developer.mozilla.org/en-US/docs/Web/CSS/animation-timeline

- **View Transitions API** — Browser-native page/element transitions. Works with MPA (multi-page) and SPA patterns.
  https://developer.mozilla.org/en-US/docs/Web/API/View_Transitions_API
